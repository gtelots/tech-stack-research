# Customers Table Documentation

## Table Overview
The `customers` table contains detailed information about all customers who have made purchases from the business. It serves as the central record for customer data, enabling personalized service, marketing analysis, and demographic insights.

## Table Structure

| Column Name    | Data Type      | Constraints        | Description                                           |
|----------------|----------------|-------------------|-------------------------------------------------------|
| customer_id    | INTEGER        | PRIMARY KEY       | Unique identifier for each customer                    |
| first_name     | VARCHAR(50)    | NOT NULL          | Customer's first name                                  |
| last_name      | VARCHAR(50)    | NOT NULL          | Customer's last name                                   |
| email          | VARCHAR(100)   | NOT NULL, UNIQUE  | Customer's email address                               |
| phone          | VARCHAR(20)    | -                 | Customer's contact phone number                        |
| address_line1  | VARCHAR(100)   | -                 | Primary address line                                   |
| address_line2  | VARCHAR(100)   | -                 | Secondary address line (apt, suite, etc.)              |
| city           | VARCHAR(50)    | -                 | City of residence                                      |
| state          | VARCHAR(50)    | -                 | State/province/region                                  |
| postal_code    | VARCHAR(20)    | -                 | ZIP or postal code                                     |
| country        | VARCHAR(50)    | -                 | Country code                                           |
| created_at     | TIMESTAMP      | DEFAULT CURRENT_TIMESTAMP | When customer record was created               |
| updated_at     | TIMESTAMP      | DEFAULT CURRENT_TIMESTAMP | When customer record was last updated          |

## Indexes
- **Primary Key**: `customer_id` (automatically indexed)
- **Secondary Index**: `idx_customers_email` - Improves query performance when searching by email and ensures email uniqueness

## Column Details

### customer_id
- **Purpose**: Serves as the unique identifier for each customer
- **Implementation**: Auto-incrementing integer (SERIAL) that ensures uniqueness
- **Business Use**: Referenced by the orders table to link transactions to customers
- **Range**: Positive integers starting from 1

### first_name
- **Purpose**: Stores the customer's given name
- **Implementation**: Variable character field with a maximum length of 50 characters
- **Business Use**: Used for personalized communication, order confirmation, and customer service
- **Constraints**: Cannot be empty (NOT NULL)

### last_name
- **Purpose**: Stores the customer's family name or surname
- **Implementation**: Variable character field with a maximum length of 50 characters
- **Business Use**: Used for personalized communication, order confirmation, and customer service
- **Constraints**: Cannot be empty (NOT NULL)

### email
- **Purpose**: Stores the customer's email address for communication
- **Implementation**: Variable character field with a maximum length of 100 characters
- **Business Use**: Primary contact method, used for order confirmations, marketing, and account identification
- **Constraints**: Cannot be empty (NOT NULL) and must be unique (UNIQUE)
- **Format**: Follows standard email format (e.g., name@domain.com)

### phone
- **Purpose**: Stores the customer's contact telephone number
- **Implementation**: Variable character field with a maximum length of 20 characters
- **Business Use**: Alternative contact method, used for order verification and delivery coordination
- **Format**: Varies by country and includes international codes in some cases

### address_line1
- **Purpose**: Stores the primary street address of the customer
- **Implementation**: Variable character field with a maximum length of 100 characters
- **Business Use**: Used for shipping, billing, and geographic analysis
- **Example Values**: "123 Main Street", "45 Park Avenue"

### address_line2
- **Purpose**: Stores additional address details such as apartment numbers
- **Implementation**: Variable character field with a maximum length of 100 characters
- **Business Use**: Provides more specific location information for deliveries
- **Example Values**: "Apt 4B", "Suite 200", "Floor 3"

### city
- **Purpose**: Stores the city or locality name
- **Implementation**: Variable character field with a maximum length of 50 characters
- **Business Use**: Used for shipping, regional analysis, and localized marketing

### state
- **Purpose**: Stores the state, province, or region
- **Implementation**: Variable character field with a maximum length of 50 characters
- **Business Use**: Used for shipping, tax calculations, and regional business intelligence
- **Format**: Often stored as standard abbreviations (e.g., "CA", "NY") in US contexts

### postal_code
- **Purpose**: Stores the ZIP or postal code
- **Implementation**: Variable character field with a maximum length of 20 characters
- **Business Use**: Used for shipping, logistics planning, and geographic targeting
- **Format**: Varies by country (e.g., 5-digit or 9-digit in US, alphanumeric in other countries)

### country
- **Purpose**: Stores the customer's country of residence
- **Implementation**: Variable character field with a maximum length of 50 characters
- **Business Use**: Used for international shipping, tax calculations, and global market analysis
- **Format**: Often stored as ISO country codes (e.g., "US", "CA", "UK")

### created_at
- **Purpose**: Records when the customer record was first created
- **Implementation**: Timestamp that defaults to the current time when record is created
- **Business Use**: Used for customer lifecycle analysis, cohort analysis, and auditing

### updated_at
- **Purpose**: Records when the customer information was last modified
- **Implementation**: Timestamp that automatically updates when the record is modified
- **Business Use**: Used for tracking data freshness, auditing, and customer profile update frequency

## Business Intelligence Applications

The customers table enables several key business analytics capabilities:

1. **Customer Demographics Analysis**: Analyzing customer distribution by location, which helps in targeted marketing and expansion planning.

2. **Customer Lifecycle Management**: Tracking how long customers have been in the system through created_at timestamps.

3. **Geographic Distribution**: Analyzing customer concentration by city, state, or country to identify market penetration.

4. **Contact Strategy Optimization**: Understanding which contact methods (email, phone) are available for different customer segments.

5. **Address Verification**: Ensuring shipping addresses are complete and valid to reduce delivery issues.

6. **Customer Data Quality Monitoring**: Tracking completeness of customer profiles over time.
