# Products Table Documentation

## Table Overview
The `products` table stores comprehensive information about all items available for sale in our business. This table serves as the central repository for product information and is crucial for inventory management, pricing strategies, and sales analysis.

## Table Structure

| Column Name     | Data Type      | Constraints        | Description                                           |
|----------------|----------------|-------------------|-------------------------------------------------------|
| product_id     | INTEGER        | PRIMARY KEY       | Unique identifier for each product                     |
| name           | VARCHAR(100)   | NOT NULL          | Product name/title                                     |
| description    | TEXT           | -                 | Detailed product description                           |
| category       | VARCHAR(50)    | -                 | Product category for classification                    |
| price          | DECIMAL(10,2)  | NOT NULL          | Current selling price                                  |
| cost           | DECIMAL(10,2)  | NOT NULL          | Purchase or manufacturing cost                         |
| stock_quantity | INTEGER        | NOT NULL          | Current inventory level                                |
| is_active      | BOOLEAN        | NOT NULL DEFAULT TRUE | Whether the product is currently active for sale   |
| created_at     | TIMESTAMP      | DEFAULT CURRENT_TIMESTAMP | When the product was added to the system        |
| updated_at     | TIMESTAMP      | DEFAULT CURRENT_TIMESTAMP | When the product was last updated              |

## Indexes
- **Primary Key**: `product_id` (automatically indexed)
- **Secondary Index**: `idx_products_category` - Improves query performance when filtering by category

## Column Details

### product_id
- **Purpose**: Serves as the unique identifier for each product in the system
- **Implementation**: Auto-incrementing integer (SERIAL) that ensures uniqueness
- **Business Use**: Referenced by the orders table to track which products are sold
- **Range**: Positive integers starting from 1

### name
- **Purpose**: Provides the official display name of the product
- **Implementation**: Variable character field with a maximum length of 100 characters
- **Business Use**: Used in product displays, invoices, and reports
- **Constraints**: Cannot be empty (NOT NULL)
- **Example Values**: "Premium Wireless Headphones", "Ergonomic Office Chair"

### description
- **Purpose**: Stores detailed information about product features, specifications, and benefits
- **Implementation**: Text field without length limit
- **Business Use**: Used for product detail pages, marketing materials, and catalog creation
- **Example Content**: "These wireless headphones feature noise-cancellation technology, 24-hour battery life, and premium sound quality with deep bass response."

### category
- **Purpose**: Classifies products into groups for easier navigation and analysis
- **Implementation**: Variable character field with a maximum length of 50 characters
- **Business Use**: Used for product filtering, category-based promotions, and inventory organization
- **Example Values**: "Electronics", "Furniture", "Clothing", "Office Supplies"

### price
- **Purpose**: Indicates the amount charged to customers
- **Implementation**: Decimal number with up to 10 digits total, 2 of which are decimal places
- **Business Use**: Used for revenue calculations, pricing strategy analysis, and customer-facing displays
- **Range**: Typically between 5.99 and 999.99 in our sample data
- **Constraints**: Cannot be null (NOT NULL)

### cost
- **Purpose**: Stores the amount paid to acquire or produce the item
- **Implementation**: Decimal number with up to 10 digits total, 2 of which are decimal places
- **Business Use**: Used for profit margin calculations, cost analysis, and inventory valuation
- **Range**: Typically 40-80% of the selling price
- **Constraints**: Cannot be null (NOT NULL)

### stock_quantity
- **Purpose**: Tracks the current inventory level
- **Implementation**: Integer representing the count of available items
- **Business Use**: Used for inventory management, low-stock alerts, and fulfillment planning
- **Range**: 0 (out of stock) to several hundred, depending on the product
- **Constraints**: Cannot be null (NOT NULL)

### is_active
- **Purpose**: Indicates whether the product is currently being sold and can be ordered
- **Implementation**: Boolean value (true/false) with a default of true
- **Business Use**: Used to control product availability without deleting product records
- **Constraints**: Cannot be null, defaults to true (NOT NULL DEFAULT TRUE)
- **Business Rules**: Inactive products cannot be included in new orders
- **Example Values**: true (product is available for sale), false (product is discontinued or temporarily unavailable)

### created_at
- **Purpose**: Records when the product was first added to the system
- **Implementation**: Timestamp that defaults to the current time when record is created
- **Business Use**: Used for auditing, historical analysis, and tracking new product additions

### updated_at
- **Purpose**: Records when the product information was last modified
- **Implementation**: Timestamp that automatically updates when the record is modified
- **Business Use**: Used for tracking product data changes, identifying recent updates, and auditing

## Business Intelligence Applications

The products table enables several key business analytics capabilities:

1. **Profit Margin Analysis**: By comparing price and cost fields, businesses can analyze profit margins across different products and categories.

2. **Inventory Valuation**: The combination of cost and stock_quantity allows for accurate inventory valuation.

3. **Stock Level Monitoring**: The stock_quantity field enables reports on low-stock items and inventory turnover.

4. **Product Performance Analysis**: When combined with the orders table, businesses can analyze which products are selling best.

5. **Category Analysis**: The category field allows for aggregated analysis of performance by product type.

6. **Pricing Strategy Optimization**: Historical pricing data (via updated_at) can be used to analyze the effect of price changes on sales.

7. **Product Lifecycle Management**: The is_active field helps track product lifecycle stages and analyze discontinuation impacts on inventory and sales.
